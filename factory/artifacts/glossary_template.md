# Glossary

**Acceptance Criteria (AC)**: Concrete checks that determine when a story is "done."

**Backpressure (per-direction)**: User-supplied artifacts (flow.md, api_spec.md, attachments) that constrain what the factory may produce for a direction.

**Direction**: A user's intent for the app, captured as a directory under `apps/<name>/directions/` plus a Direction Tracker GitHub issue.

**Story**: One implementation unit of work. Lives at `stories/<n>-<slug>.md`.
