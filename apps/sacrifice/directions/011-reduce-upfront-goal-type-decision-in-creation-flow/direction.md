# Reduce upfront goal-type decision in creation flow

## Why

The current flow inserts an extra required classification step before goal entry, which increases effort at the start of the core task.

## Acceptance Criteria

- [ ] A user can begin creating a goal without first selecting among the four internal goal types.
- [ ] The creation flow still maps the final submission to one supported backend goal type.
